## Game Statistics

### Purpose

The backend tracks two aggregate metrics for display in the frontend:

- **Total games played** – the number of games that have reached a meaningful state.
- **Currently active games** – the number of games in progress right now.

These statistics are anonymous counters with no connection to individual users. No personal data is collected or stored for this feature.

### Minimum Move Threshold

A game is only counted toward statistics once it has reached **10 moves** (total moves across all players). This filters out accidental starts, abandoned setups, and brief test sessions.

### Database Schema

```sql
CREATE TABLE game_stats (
    game_id    VARCHAR(36) PRIMARY KEY,  -- UUID generated by the frontend
    status     ENUM('active', 'completed', 'abandoned') NOT NULL DEFAULT 'active',
    move_count INT UNSIGNED NOT NULL DEFAULT 0,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### API Endpoints

| Method  | Path                       | Description                                      |
|---------|----------------------------|--------------------------------------------------|
| `POST`  | `/api/games`               | Register a new game session (status: `active`)   |
| `PATCH` | `/api/games/{id}/move`     | Increment the move counter by 1                  |
| `PATCH` | `/api/games/{id}/end`      | Set status to `completed` or `abandoned`         |
| `GET`   | `/api/games/stats`         | Return aggregated statistics                     |

#### Response Format: `GET /api/games/stats`

```json
{
  "totalGames": 1423,
  "activeGames": 7
}
```

- `totalGames` counts all games (any status) with `move_count >= 10`.
- `activeGames` counts games with status `active` and `move_count >= 10`.

### Abandoned Game Detection

When a player closes the browser or loses connection, no explicit end request is sent. Games that remain in `active` status without receiving a move update for an extended period are considered abandoned. The specific detection strategy (server-side cronjob vs. time-window filter at query time) is defined in the feature specification.

Remote multiplayer will allow players on different devices to participate in the same game over the internet. This feature is not yet specified.
